Aprendiendo PowerBI

  • DAX(Data Analysis Expressions): es el lenguaje de consulta de Powert Bi y tambien se pued hacer consulta a traves de query editor
  • Buenas practicas de Modelado de Datos *
    • Entidad de Negocio = Tabla Tablas => Existen 2 tipos: * Fact(hechos): Eventos Medidos *Dim(Dimensiones): Utilizados para segmentar tablas [que, quien, como, cuando, donde]
  • Las tablas se pueden estructura en estrella o de forma horizontal, siendo el fact el principal que se relacionara con el resto de tablas
  • A la hora de actualizar en Refresh se actualizara todas las tablas, para que eso no suceda debes de darle en TransformData y luego click derecho en la tabla que no quieres que se refresque
  • Agregar Measure(medida) = agregar calculo
  • Para mover los measures creados debemos de hacerlo desde la parte de Model, seleccionandolo y arrastrandolo a la tabla nueva
  • BLANK() =>Me permite poner una posicion en blanco
  • Funciones de Fecha
    • Mes = MONTH(nombreTabla[nombreCampo])
    • Trimestre = QUARTER(nombreTabla[nombreCampo])
    • Año = YEAR(nombreTabla[nombreCampo])
    • Dia = DAY(nombreTabla[nombreCampo])
    • numeroSemanaAño = WEEKNUM(nombreTabla[nombreCampo])
    • numeroDiaSemana = WEEKDAY(nombreTabla[nombreCampo])
  • Funciones de Logicas y Condicionales
    • nombreColumna = IF(nombreTabla[nombreCampo]>2,"Es mayor 2","No es mayor 2")
    • FinSemana = IF(OR(DimCalendar[diaSemana]=6,DimCalendar[diaSemana]=7),"Fin de Semana","Dia Laboral") =>Tambien es OR() = ||
    • FinSemana = IF(DimCalendar[diaSemana] in {6,7},"Fin de Semana","Dia Laboral")
    • MalaSuerte = IF(DimCalendar[CalendarDayOfWeekLabel]="Tuesday" && DimCalendar[Dia]=13,"Mala Suerte","Dia Normal")
  • Relaciones de Texto(Concatenacion)
    • UbicacionCompleta = DimGeography[RegionCountryName] & " " &DimGeography[ContinentName]
    • Trimestre = "Trimestre N°" & QUARTER(DimCalendar[DateKey])
    • MesCorto = LEFT(DimCalendar[CalendarMonthLabel],3) =>Toma los primeros 3 caracteres
    • UbicacionCompleta = UPPER(DimGeography[RegionCountryName])
    • UbicacionCompleta = LOWER(DimGeography[RegionCountryName])
    • Tienda = SUBSTITUTE(DimStores[StoreName],"Store","Tienda")
  • Funcion Related y Matematica
    • Ubicacion Fisica = RELATED(DimGeography[UbicacionCompleta]) =>Asi traemos datos de otra tabla relacionada
    • PU Promedio = AVERAGE(DimProduct[UnitPrice])
  • Funcion Contar
    • Cantidad Tiendas = COUNT(DimStores[StoreKey]) =>Cuenta todo tipo de registro
    • Cantidad Tiendas = COUNTA(DimStores[StoreKey]) =>Cuenta solo registros llenos no en blanco
    • Cantidad Tienda con Ventas = DISTINCTCOUNT(FactSales[StoreKey]) =>Cuenta solo registros llenos no en blanco
    • Cantidad Regiones en Blanco = COUNTBLANK(DimGeography[RegionCountryName]) =>Cuenta solo registros en blanco
    • Cantidad Regiones 2 = COUNTROWS(DimGeography) =>Cuenta la cantidad de filas solo de tablas
  • Funcion Calculate
    • Total Ordenes Devuelta = CALCULATE([Total Ordenes],FactSales[ReturnQuantity]>0) =>total ordenes es un measure
  • Funcion ALL y Filter
    • All Total Ordenes Globales = CALCULATE([Total Ordenes],ALL(FactSales)) =>El all no toma en cuenta los filtros
    • Total Ordenes Devueltas Multiples Filter = CALCULATE([Total Ordenes],FILTER(FactSales,FactSales[TipoDevolucion]="Multiple")) =>Filter solo devuelve valores tomando en cuenta el filtro el resto sale n blank lo ignora y
